================================================================================
SMOKE TEST - SISTEMA DE GESTAO DE HOTEIS
================================================================================

Este script descreve os casos de teste para o smoke test do sistema.
Cada caso de teste contempla um cenario de sucesso para cada funcionalidade.

IMPORTANTE: Execute os testes na ordem apresentada, pois alguns dependem de dados
criados em testes anteriores.

================================================================================
PREPARACAO
================================================================================

1. Compilar o projeto:
   - Via Code::Blocks: Abrir TP1.cbp e compilar (F9)
   - Via Makefile: Executar "make" no diretorio raiz
   - Via linha de comando: g++ -std=c++11 -Wall -Iinclude -c src/*.cpp -o bin/sistema_hoteis.exe

2. Executar o sistema: bin/sistema_hoteis.exe

TESTE 25: VALIDACAO DE CONFLITO DE RESERVAS (Cenário de Sucesso)
================================================================================

Objetivo: Verificar que o sistema detecta conflitos de datas em reservas

Passos:
1. Criar um hotel e um quarto (se necessário)
2. Criar uma reserva para o quarto:
   - Data Chegada: "01-FEB-2024"
   - Data Partida: "05-FEB-2024"
3. Tentar criar outra reserva para o mesmo quarto com datas conflitantes:
   - Data Chegada: "03-FEB-2024"
   - Data Partida: "07-FEB-2024"
4. Verificar que o sistema impede a criação (mensagem de erro)
5. Criar uma reserva sem conflito:
   - Data Chegada: "10-FEB-2024"
   - Data Partida: "15-FEB-2024"
6. Verificar que esta reserva é criada com sucesso

Resultado Esperado: Sistema detecta e impede conflitos de datas corretamente

RESUMO DOS CASOS DE TESTE

Total de casos de teste: 25

Funcionalidades testadas:
- Autenticação: Criar conta, Login
- Gerente: Atualizar perfil

- Hóspedes: Criar, Listar, Atualizar, Buscar, Excluir
- Persistência: Verificação de dados entre execuções
- Integridade: Validação de exclusões em cascata

Todos os casos de teste contemplam cenários de sucesso.

================================================================================
OBSERVAÇÕES
================================================================================

- Execute os testes na ordem apresentada
- Alguns testes dependem de dados criados em testes anteriores
- Os dados de exemplo podem ser alterados, mas devem seguir os formatos válidos
- Para o teste de persistência, certifique-se de que o diretório "data/" existe
- O sistema deve compilar sem erros antes de executar os testes
